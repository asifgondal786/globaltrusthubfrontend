# Contributing to GlobalTrustHub Frontend

Thank you for your interest in contributing to GlobalTrustHub! We welcome contributions from the community to help us build a trusted digital ecosystem for global education and work.

## Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally:
    ```bash
    git clone https://github.com/YOUR_USERNAME/globaltrusthubfrontend.git
    cd globaltrusthubfrontend
    ```
3.  **Install dependencies**:
    ```bash
    flutter pub get
    ```
4.  **Create a branch** for your feature or fix:
    ```bash
    git checkout -b feature/amazing-feature
    ```

## Development Workflow

-   **Code Style**: We follow standard Dart and Flutter linting rules. Ensure your code passes analysis before committing.
-   **Testing**: Please add tests for new features. customized logic should be verified.
-   **Commits**: Use clear, descriptive commit messages.
    -   Example: `feat: add dark mode support to dashboard`
    -   Example: `fix: resolve login api cors issue`

## Submission Guidelines

1.  Push your changes to your fork.
2.  Submit a **Pull Request (PR)** to the `main` branch.
3.  Provide a clear description of what changed and why.
4.  Link to any relevant issues.

## Reporting Issues

If you find a bug or have a feature request, please open an issue in the repository with details on how to reproduce it.

Thank you for helping us mitigate fraud and empower global citizens!
